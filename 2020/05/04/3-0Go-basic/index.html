goroutine基本模型和调度设计策略

进程 按照时间轴 顺序执行

CPU轮询调度 一个进程不能超过时间片

切换进程的产生了切换成本
进程/线程数量越多，切换成本越大，也就是越浪费
CPU 100% 60%执行程序 40%切换

高消耗调度CPU,高内存占用。

线程 用户空间和内核空间

  • 用户空间内，协程s通过协程调度器绑定线程thread,N比1
  • 用户空间内，协程1通过协程调度器绑定线程thread,1比1
  • 用户空间内，协程s通过协程调度器绑定内核空间里的线程thread,M比N

调度器的设计策略

  • 复用线程
  • 利用并行
    • GOMAXPROCS限定P的个数 = CPU核数/2

  • 抢占

  • 全局G队列

29 chnnel有缓冲与无缓冲同步问题

无缓冲的，就是channel中没有存储goroutine ，所以一方开启goroutine的时候，会进行锁住。

有缓冲的，就是channel中存储goroutine ，不会锁住

当channel已经满，再往里面写入，就会阻塞。
当channel为空，从里面取数据，也会阻塞。
